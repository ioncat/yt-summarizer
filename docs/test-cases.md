# Test Cases — YT Summarizer

Функциональные тест-кейсы для регрессионного тестирования критических поведений системы.  
Формат: Given / When / Then. Статус заполняется при каждом прогоне вручную.

---

## TC-001 · Язык заголовков глав соответствует языку видео

**Область**: subtitle_extractor · chapter source priority  
**Приоритет**: 🔴 P0 — затрагивает весь pipeline (formatted_text → cleaned_text → summary → mindmap)  
**Добавлен**: 23.05.2026  
**Причина**: YouTube API переводит `info["chapters"]` на английский независимо от кук,
Accept-Language и player_client. Подтверждено диагностикой (`tools/debug_chapters.py`, 6 конфигураций).

---

### TC-001-A: Русское видео с таймкодами в описании

**Preconditions**:
- Видео на русском языке
- Автор указал таймкоды в описании видео (список глав с временными метками)
- Пример: `https://www.youtube.com/watch?v=RagM_T1HCuo`

**Given** пользователь извлекает субтитры русскоязычного видео  
**When** экстракция завершена  
**Then**:
- Заголовки глав в `formatted_text` на русском языке (`## Хаос и бег по минному полю`)
- Заголовки глав в `cleaned_text` на русском (LLM их сохраняет без изменений)
- Заголовки глав в `summary_text` на русском
- Заголовки в mindmap на русском
- В серверном логе НЕТ строк `[CHAPTER_SOURCE]`

**Fail criteria**:
- Любой заголовок на английском языке (`## Chaos and running through a minefield`)
- Присутствие `[CHAPTER_SOURCE] ... falling back to YouTube API chapters` в логе

---

### TC-001-B: Видео без таймкодов в описании → fallback на YouTube chapters

**Preconditions**:
- Видео на любом языке
- Описание не содержит таймкодов (или содержит меньше 2 временных меток)
- `info["chapters"]` от YouTube существует

**Given** описание видео не содержит таймкодов  
**When** экстракция завершена  
**Then**:
- Главы взяты из `info["chapters"]` (могут быть на английском — это ожидаемое поведение)
- В логе есть `[CHAPTER_SOURCE] ... falling back to YouTube API chapters (may be translated)`
- Форматирование не сломано, текст разбит по главам

**Fail criteria**:
- Исключение при построении метаданных
- Текст не разбит на главы при наличии `info["chapters"]`

---

### TC-001-C: Видео без глав вообще → gap-based fallback

**Preconditions**:
- Видео без авторских таймкодов в описании
- `info["chapters"]` пуст или отсутствует

**Given** у видео нет ни description-таймкодов, ни YouTube-глав  
**When** экстракция завершена  
**Then**:
- `formatted_text` разбит по 4-секундным паузам (gap-based, существующая логика)
- `has_chapters = False` в результате
- В логе НЕТ `[CHAPTER_SOURCE]`
- В UI нет `##`-заголовков

**Fail criteria**:
- Присутствие пустых `## ` заголовков в тексте
- Ошибка экстракции

---

### TC-001-D: Детекция языкового расхождения

**Preconditions**:
- Видео на русском (`language = "ru"`)
- Главы взяты из `info["chapters"]` (английские — fallback-кейс)

**Given** язык субтитров русский, заголовки глав на латинском скрипте (английские)  
**When** экстракция завершена  
**Then**:
- В логе есть `[CHAPTER_SOURCE] ... chapter titles are in latin script but subtitle language is 'ru' (expected cyrillic)`
- Предупреждение содержит `video_id` для идентификации

**Fail criteria**:
- Предупреждение не появляется при реальном расхождении языков

---

### TC-001-E: Английское видео — заголовки на английском, предупреждений нет

**Preconditions**:
- Видео на английском языке (`language = "en"`)
- Описание содержит английские таймкоды

**Given** пользователь извлекает субтитры англоязычного видео  
**When** экстракция завершена  
**Then**:
- Заголовки глав на английском — это правильно
- В логе НЕТ `[CHAPTER_SOURCE]` предупреждений (english = latin = совпадение)

**Fail criteria**:
- Ложное срабатывание `[CHAPTER_SOURCE]` для англоязычных видео

## Как запускать диагностику вручную

```bash
# Проверить главы из разных источников для любого видео:
python tools/debug_chapters.py "VIDEO_URL"

# С явным путём к cookies:
python tools/debug_chapters.py "VIDEO_URL" "app/data/cookies.txt"
```

Скрипт прогоняет 6 конфигураций yt-dlp и отдельно выводит таймкоды из описания.  
Ожидаемый результат для русского видео: все 6 конфигураций → английские главы, блок "Description timecodes" → русские.
