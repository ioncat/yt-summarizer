# Effort Log ‚Äî yt-summarizer

Time analysis based on git history (commit timestamps).

## Methodology

Time is estimated by **commit clusters**, not by calendar day. This handles
multiple sessions per day (e.g. morning + evening work) with reasonable accuracy.

**Rules:**

1. **Cluster** = a sequence of commits with gaps ‚â§ 1 hour between consecutive commits.
   A gap > 1 hour ends the cluster and starts a new one.
2. **Session time** = timestamp of last commit in cluster ‚àí timestamp of first commit in cluster.
3. **Single-commit cluster** ‚Üí estimated at **~15 min** (minimum).
4. Multiple clusters in one day ‚Üí multiple rows in the table.
5. Time spent **before the first commit** (planning, discussion) and **after the last commit**
   (testing, debugging) is NOT counted. The number is therefore a lower bound on real time.

**Why this works:** cluster-based estimation reflects actual work cadence without
manual tracking. Git history is the source of truth.
